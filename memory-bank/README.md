# Memory Bank

This directory contains project documentation and context for the Live Scotland Yard project.

## Quick Reference

**For Ralph Loop**: Start with `../PROMPT.md`, which references `CONTEXT.md` for full project context.

## Document Structure

### Core Documents (Active)

- **CONTEXT.md** - Consolidated reference for all project context
  - Overview, architecture, tech stack
  - Game rules summary
  - Phase 1 implementation details
  - Phase 2 requirements
  - Code style and patterns

- **progress.md** - Task tracker and completion status
  - What's done, what's in progress, what's next
  - Detailed breakdown of all phases

- **productContext.md** - Product vision and requirements
  - Feature specifications
  - Engine API design
  - Technical stack details

- **activeContext.md** - Current goals and blockers
  - Near-term objectives
  - Immediate next steps

### Reference Documents

- **projectBrief.md** - High-level project purpose and goals
- **systemPatterns.md** - Design patterns and type mappings
- **architect.md** - Architectural decisions (if exists)
- **decisionLog.md** - Record of key decisions made

### Archive (Reference Only)

- **GITHUB_RESEARCH.md** - Research on Scotland Yard implementations
  - Evaluated 6+ repositories
  - Selected `ZikZhao/Scotland-Yard-2024` as canonical engine
  - Detailed comparison and rationale

- **IMPLEMENTATION_PLAN.md** - Original detailed implementation plan
  - Phase 0-4 breakdown
  - Now superseded by progress.md for task tracking

## Usage

### For Ralph Loop

1. Read `../PROMPT.md` for current task
2. Reference `CONTEXT.md` for full project context
3. Check `progress.md` for detailed task breakdown
4. Consult other docs as needed for specific details

### For Human Developers

1. Start with `CONTEXT.md` for complete overview
2. Check `progress.md` to see current state
3. Review `productContext.md` for feature details
4. Consult archive docs for historical context

## Maintenance

- **CONTEXT.md** should be updated when architecture changes
- **progress.md** should be updated as tasks complete
- Other docs can be updated less frequently
- Archive docs are read-only reference material
